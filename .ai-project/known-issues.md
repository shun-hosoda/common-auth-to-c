# 既知の落とし穴・自己診断コマンド

## 認証基盤でよくある落とし穴

### セキュリティ

| 問題 | 対策 |
|------|------|
| パスワードの平文保存 | bcrypt/argon2 でハッシュ化必須 |
| JWT秘密鍵のハードコード | 環境変数 or シークレット管理サービス |
| トークンの無期限有効 | アクセストークン短命 + リフレッシュトークンでローテーション |
| CSRF対策漏れ | SameSite Cookie + CSRF Token |
| レート制限未実装 | ログイン試行・API呼び出しにレート制限 |
| メールアドレス列挙攻撃 | 登録/リセットで「アカウント存在有無」を漏らさない |

### データ整合性

| 問題 | 対策 |
|------|------|
| ソーシャルログインとメール紐付け不整合 | メールアドレスベースでアカウントリンク戦略を定義 |
| セッション無効化漏れ | パスワード変更時・退会時に全セッション破棄 |
| 退会済みユーザーの再登録 | ソフトデリート + 再登録ポリシーの明確化 |

### パフォーマンス

| 問題 | 対策 |
|------|------|
| DB直接参照でのセッション検証 | Redis等キャッシュの活用 |
| JWTの無効化不能 | ブラックリスト or 短命トークン + リフレッシュ |
